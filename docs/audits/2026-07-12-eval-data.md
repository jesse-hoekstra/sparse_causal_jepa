# Fidelity audit: eval harness + bounce environment vs Baumgartner et al. (sources/dynamical_system.pdf)

Date: 2026-07-12. Scope: report only, no code edited. Context trusted: two 300k-step runs, MCC
0.01–0.03, path matrix pruned to identity, shd_param frozen at 1.4595 in both runs.

All paper citations are to `sources/dynamical_system.pdf` (PMLR 323, CLeaR 2026); page numbers are
0-indexed PDF pages as extracted (pdf p. N = printed page N+1).

## 1. `src/scjepa/eval/parameters.py` vs App. F.1 (pdf pp. 38–39)

Their spec (verbatim): "fitting a small MLP onto every combination of marginalised predictions and
marginalised ground truth parameters, θ_i ≈ MLP_ij(θ̂_j) … MCC = (1/I) Σ_i max_j (R²_ij)" and "a
one-hidden-layer MLP with a hidden dimension of 32 and 5,000 sampled training points, as well as
cross-validation with a 10% and 90% split."

| Item | Theirs | Ours (`parameters.py`) | Verdict |
|---|---|---|---|
| Regression direction | true θ_i regressed FROM learned θ̂_j | `x = learned[:, j]`, `y = target[:, i]`, fit `mlp(x)≈y` (lines 94–114) | **Match.** This is the direction that "enforces surjectivity"; reversed would be a different metric. |
| MLP | 1 hidden layer, width 32 | `nn.Linear(1,32), Tanh, Linear(32,1)` (line 103) | Match (activation unspecified upstream; Tanh is a reasonable pick — cosmetic). |
| Samples / split | 5,000 training points, 90/10 CV | `max_samples=5000` TOTAL, then 90 train /10 val (lines 88–101) | Near-match: ours trains on 4,500 when 5,000 are available. Cosmetic. |
| Optimization | "converges quickly" (unspecified) | Adam 1e-2, 300 full-batch epochs | Unspecified upstream; oracle below confirms convergence. |
| Negative R² | unspecified | clamped at 0 (`_r_squared`, lines 57–65, documented) | Can only RAISE MCC (and only when all J dims are negative, since max_j is taken). Cannot hide a recoverable representation. Cosmetic. |
| Sample unit | per-trajectory sample; I = 5 masses, J = latent dims of θ̂ | pooled (episode, object) samples; I = 1 (mass), J = d per-slot dims | **Deliberate reformulation, stricter than theirs — see below.** |

**Pooling scoping.** Their encoder emits one θ̂ per trajectory containing all 5 masses; the
mean-max over the full I×J grid credits ANY latent dim for ANY mass (arbitrary permutation).
Our pooled convention requires the SAME learned dim of slot i to encode object i's mass via the
SAME map for every slot. Two consequences, both in the conservative direction:

- It cannot credit a representation where slot i encodes m_{π(i)} or where the slot→mass map is
  slot-dependent. Oracle Case C (per-slot PERFECT linear map but slot-dependent sign): pooled
  nonlinear MCC = **0.091** where their treatment would score ~1. However, this failure mode is
  architecturally precluded here: pooling weights are shared across slots and
  `CrossSlotAttnPooling` deliberately carries no slot-identity PE (D14), so a systematic
  slot-index-dependent map has no parameters to live in. Severity: scoping caveat, not a bug —
  and it can only under-report, never inflate.
- Leakage note: (episode, object) samples from one episode land in both train and val of the MLP
  split; with cross-slot pooling the learned values within an episode are correlated. This can only
  INFLATE MCC slightly, so it cannot explain an apparent failure.

**Verdict: faithful implementation of F.1**; the one real deviation (pooled samples) is stricter
than the paper and cannot floor an otherwise-recoverable, weight-shared representation.

## 2. Oracle results (metric sanity, run 2026-07-12, scripts in this scratchpad)

`oracle_mcc.py`, S = 2560 = 512 eval episodes × 5 balls, d = 32, masses ~ clamped N(1.5, 0.5):

| Case | nonlinear_mcc | mean_max_correlation |
|---|---|---|
| A: learned dim 7 = tanh(1.7(m−1.5)) + 0.02·noise, 31 noise dims | **0.974** | 0.977 |
| B: pure noise, S = 2560 (final-eval size) | **0.000** | 0.035 |
| B′: pure noise, S = 640 (periodic-eval size, 128 episodes) | 0.024 | 0.138 |
| C: per-slot perfect, slot-dependent sign (pooled) | 0.091 | 0.219 |

Ceiling ≈ 1 and floor ≈ 0 confirmed. **The observed run MCC of 0.01–0.03 is numerically
indistinguishable from the pure-noise floor at both eval sizes** — the eval is reporting a
genuinely mass-free Ŝ^ph, not masking a recoverable one. (Also note D15's caveat holds: at the
periodic-eval size of 640 samples the noise floor is ~0.02–0.03, so readings in that band mean
zero.)

## 3. Graph readout + D15 transition alignment (`graph.py`, `harness.py`)

- **Contact indexing:** `simulate_bounce` records into `contacts[step]` during the substeps that
  advance `states[step] → states[step+1]` (bounce.py:88–110), so `contacts[t]` IS transition
  t→t+1. D15 predictions cover t = th−1 … L−2; the harness takes `contacts[:, th-1:]`
  (harness.py:97–101) — **exact one-to-one alignment**, K = L − th entries on both sides,
  flattened in the same (B, K) order as the model's flattened outputs
  (`repeat_interleave`/`flatten(0,1)` in state_jepa.py:73–83). No off-by-one. As scoped in the
  task: this only affects SHD, and it is correct anyway.
- **GT conventions** (graph.py:26–47) match the physics and D11's documented convention; the Fig. 2
  bounce DAG (pdf p. 7: m_1..m_5 → o_1..o_5 edges "only active in subsets of the state space")
  is the same local-graph-on-contacts idea.
- **Learned readout** (graph.py:50–62): token layout [state | params | aux] matches the Spartan
  module; path product `(A_L+I)···(A_1+I)` built left-appending (spartan.py:276–280) matches
  SPARTAN Eq. 5 / their Eq. 7; orientation Ā_ij = #paths j→i with q_i·k_j logits matches;
  threshold `>= 0.5` on integer path counts ≡ the paper's "≥ 1".
- **Independent cross-check that the readout is honest:** from the measured collision statistics,
  E[GT param edges per transition] = 5·P(ball involved) + 2·E[contact pairs]
  = 5·0.1425 + 2·0.371 = **1.455**, and SHD against an all-zero learned param block is exactly the
  GT edge count. The forensics' frozen shd_param = **1.4595** reproduces this to within sampling
  error — confirming both that the harness computes SHD correctly and that the runs truly learned
  zero param edges.

**One real GT-graph discrepancy (G1).** Under `radius_from_mass=True` (the baumgartner config),
wall reflections trigger at `pos < r_i` with r_i ∝ m_i (bounce.py:93–95), so ∂x_{t+1}/∂m_i ≠ 0 at
wall-bounce transitions — yet `param_graph` grants mass edges only at ball-ball contacts
(graph.py:44–47), and the bounce.py docstring's "mass-independent wall reflections" is false in
this variant (timing/geometry is mass-dependent; only the velocity-update rule is not). Under the
paper's Jacobian-style local-graph semantics these are missing GT edges. Measured: 19.2% of
(episode, ball) context windows contain a wall bounce without any own ball-contact.
**Severity: distorts-metrics (shd_param only, baumgartner config only; MCC unaffected).** It also
means a model that (correctly!) kept a self param edge at wall bounces would be penalized in SHD.

## 4. `src/scjepa/data/bounce.py` vs App. E "Bounce" (pdf p. 38)

Their spec (verbatim): "five round balls constrained in a square box, observable through
object-centric representations. Each ball can collide elastically … governed by the mass of each
object, which is the dynamical factor of variation, sampled from a non-zero mean normal
distribution. For ease of observation, the radius of each ball is also proportional to its mass."

| Item | Theirs | Ours | Verdict |
|---|---|---|---|
| 5 balls, square box, elastic ball & wall collisions | ✓ | ✓ (impulse formula verified: Δv_i = 2 m_j/(m_i+m_j)(v_rel·n)n; e = 1; no tunneling at speed 0.7, sub_dt 0.0125) | Match |
| Masses ~ non-zero-mean normal | params unspecified | N(1.5, 0.5) clamped [0.5, 3] | Match up to unspecified params (flagged in config; D13 email item) |
| Radius ∝ mass | proportional to ITS mass (constant unspecified, presumably fixed) | `radii = 0.08 · m_i / mean(m)` (bounce.py:221) — **per-episode normalization** | **Discrepancy G2, see below** |
| Observation | object-centric, token content unspecified (D13's open question) | states regime: [x, y, vx, vy] | D13 flag stands; radius NOT in our token |
| Episode length | "longer sequences" (conv encoder because of it, pdf p. 8); length unspecified | L = 40, dt = 0.1 | Plausible-match; unspecified upstream |
| Initial conditions | unspecified | rejection-sampled with MEAN-radius margins → with per-ball radii, **72/300 episodes start overlapping** (ball-ball or wall) | Cosmetic-to-minor: physics stays self-consistent and contacts are recorded, just extra early collisions |

**G2 — the radius normalization caps MCC at ~0.77 (distorts-metrics at the top end).**
Physics fact: elastic ball-ball impulses depend only on mass RATIOS, and wall reflections'
velocity rule is mass-independent — so absolute mass is unidentifiable from dynamics alone; the
only absolute-scale channel is geometry through the radius. Their fixed r_i ∝ m_i makes contact
distances and wall turning-points reveal absolute m_i. Our per-episode normalization makes
geometry reveal only m_i/mean(m). Oracle: planting the EXACT scale-free representation
m_i/mean(m) yields nonlinear MCC **0.775** (Pearson 0.889) vs 0.9995 for exact m_i. So even a
perfect model of our current baumgartner config cannot reach their Fig. 3 reference (~0.9);
fix is a fixed proportionality constant (with placement/margins updated per-ball).
**Irrelevant to the current failure** (0.01 vs ceiling 0.77), but it must be fixed before their
number is a meaningful target.

**Collision statistics (300 episodes, baumgartner config: L=40, N=5, speed 0.7, ctx=10;
`collision_stats.py`):**

| Statistic | Value |
|---|---|
| P(≥1 ball-ball contact in a transition) | 0.321 |
| Mean contact pairs per transition | 0.371 |
| P(episode has ≥1 collision in the 10-step context window) | **0.973** |
| P(ball has ≥1 OWN collision in the 10-step context) | **0.767** |
| P(own collision OR wall bounce in context) — wall bounces are mass-evidence under radius∝mass | 0.959 |
| P(arbitrary 10-step window contains ≥1 collision) | 0.981 |
| Fraction of predicted transitions (t = 9…38) with ≥1 contact | 0.323 |
| Fraction of predicted transitions involving a given ball | **0.1425** (≈ 4.3 of K = 30 per episode) |

**The "windows contain no collisions → parameters unidentifiable by physics" hypothesis is
quantitatively rejected**: 97% of context windows contain a collision, 77% of (episode, ball)
samples have own-collision evidence (96% counting wall geometry). Physics leaves the ceiling at
≈ 0.77·(own-collision coverage adjustments) — nowhere near the observed 0.01.

**Mass signal in the single-step objective (`mass_signal.py`).** Mass-blind counterfactual = one
simulator step from the same x_t with all masses set to the episode mean (⇒ equal radii). Excess
MSE of the best-possible mass-blind predictor over the D15 predicted transitions:

| Quantity | Value |
|---|---|
| Excess MSE per dim [x, y, vx, vy] | [1.4e-4, 1.5e-4, 5.3e-2, 5.4e-2] |
| State variance per dim | [0.063, 0.061, 0.265, 0.271] |
| Variance-normalized excess, mean over 4 dims | **0.101** (= 1.7 × τ = 0.06) |
| … using position dims only | **0.0023** (= 0.04 × τ) |
| Excess MSE given ball in contact / not in contact | 0.073 / 0.019 (non-contact share is radius-geometry: wall timing + counterfactual contact onset) |

Interpretation: if the jointly-trained target embedding represented the 4 state dims with equal
unit-variance weight, a mass-blind (zero-param-edge) model could NOT satisfy τ = 0.06 — the
constraint would force param edges. But nothing in the objective pins the target's content: a
target embedding that reallocates toward position (velocity down-weighted) has a mass-blind
normalized floor of 0.0023, making zero-param-edge constraint satisfaction trivial. This is the
content-level sibling of D12's scale collapse and is the leading account consistent with both runs
pruning to identity WHILE satisfying the constraint. It is a training/objective issue, not an eval
issue. Actionable diagnostic on existing checkpoints: measure the learned target embed's
per-state-dim Jacobian/weight norms (velocity vs position content).

## 5. Their training/eval protocol (task 5) — exact citations

- Intro (pdf p. 1): "encodes observed trajectories into latent parameters and then decodes them,
  **together with the initial state of the system, to reconstruct future trajectories** (figure 1)."
- §3.1 (pdf p. 5): "the decoder, p_ψ(τ|θ̂, x₀), **generates the reconstructed trajectory
  conditioned on an initial state x₀**. In practice, conditioning on the initial state is
  implemented by concatenating the latent parameter with the initial state."
- App. B.4 (pdf p. 24, Lemma 1 machinery): "**Under autoregressive rollouts, the model reuses the
  same inferred latent parameter θ̂ to decode the dynamics at every step along the trajectory.**"
- Fig. 4 caption (pdf p. 9): "**Autoregressive rollouts** … produced from the learnt (a) SPARTAN
  … models."
- Eqs. 8 and 11 take expectations over x_t ∈ τ ∈ D — SPARTAN is applied at every state along the
  trajectory; Eq. 9's constraint is on the WHOLE-trajectory L_rec + L_KL + L_logit ≤ L*.
- §4 Baselines (pdf p. 8): bounce encoder is convolutional, "more efficient for **longer
  sequences**" — their bounce trajectories are long; exact length, L*, λ_logit unspecified
  (D13 email items confirmed still open).

Whether training reconstruction is teacher-forced per-step or free-running is not stated
explicitly, but the decoder's ONLY inputs are (x₀ or x_t, θ̂) and the trajectory-level likelihood
p(τ|θ̂, x₀); the theory is explicitly premised on autoregressive rollouts. Either reading gives
the same structural conclusion:

**Implication for ours.** In their objective, a wrong mass corrupts every state after the first
collision of a (long) trajectory — under rollout the position error compounds for the remaining
horizon, and even teacher-forced, every post-collision transition of a long sequence re-exposes
the impulse error inside a constraint that covers the WHOLE trajectory. A mass-blind decoder can
therefore never satisfy their L*: **their constrained optimization structurally forces param
edges.** Our D15 objective is strictly teacher-forced single-step: the mass signal lives inside
one dt, on the 14.25% of (ball, transition) events with contact, worth 0.101 in normalized units
at best and 0.0023 if the (free) target content drifts to positions. Additionally their θ̂
encoder sees the full trajectory (own-collision evidence ≈ 1 − (1−0.1425)^39 ≈ 0.998 of samples)
vs our 10-step context (0.767, or 0.959 with wall geometry). So yes — **our setup faces a
fundamentally weaker mass-identification signal than theirs**, and unlike theirs, ours has an
escape hatch (target content reallocation) through which a well-optimized run satisfies the
constraint with zero param edges.

## Verdicts

**(a) Is the eval layer trustworthy? YES.** The metric matches F.1 (right regression direction,
right MLP, right split), oracles at 0.974 ceiling / 0.000–0.024 floor; the D15 transition
alignment is exact; and the frozen shd_param value independently reproduces, from first
principles, exactly what a zero-param-edge path matrix must score — the eval is measuring the
model correctly, and the observed MCC 0.01–0.03 is the genuine noise floor of a mass-free Ŝ^ph.
Nothing in the measurement layer can be hiding a recoverable representation. Caveats: shd_param
semantics are slightly off under radius_from_mass (G1), and MCC has a hard ceiling of ~0.77 under
the current radius normalization (G2) — both matter only once MCC lifts off the floor.

**(b) Collision statistics (baumgartner config, L=40, ctx=10, speed 0.7):** 32% of transitions
contain a collision (0.37 pairs/transition); 97.3% of 10-step context windows contain ≥1
collision (98.1% for arbitrary windows); 76.7% of (episode, ball) samples have own-collision
evidence in the context (95.9% incl. mass-revealing wall bounces); 14.25% of predicted
transitions involve a given ball (~4.3 of 30 per episode). **Physics does not starve
identifiability** — the over-loose-window hypothesis is rejected; the quantified weak point is
the per-transition INCENTIVE (mass-blind excess = 0.101 normalized, and only 0.0023 through
position dims), not the availability of evidence.

**(c) What their protocol implies for ours:** their decoder reconstructs whole trajectories from
(x₀, θ̂) autoregressively, with the sparsity constraint over the full-trajectory loss — mass
errors compound and a mass-blind model cannot satisfy their L*, so param edges are structurally
forced. Our teacher-forced single-step JEPA concentrates the mass signal in ~14% of transitions
within one dt AND leaves the target-embedding content free to drift toward the mass-independent
position dims, at which point τ = 0.06 is satisfiable with zero param edges. That combination —
not the eval, and not collision scarcity — is the leading structural explanation for both runs
pruning to identity. Concrete follow-ups outside this audit's scope: (i) inspect checkpoints'
target-embed velocity content; (ii) consider multi-step / rollout prediction or a
velocity-preserving constraint on the target; (iii) fix G2 (fixed radius constant) before
treating their Fig. 3 ~0.9 as the target; (iv) their L*, λ_logit, trajectory length remain
unspecified — the D13 email is worth sending.
